# Act As User

Use this skill to evaluate implemented work as a real user, not as a code reviewer.

Workflow:
1. Read `docs/general_idea.md`, `docs/general_tasks.md`, and `docs/development_logs.md`.
2. Identify the completed feature or current iteration scope.
3. List the user goals affected by the work.
4. Walk each journey step by step: route, UI action, data entry, expected result.
5. Flag missing states, confusing flows, incomplete outcomes, or gaps between the product idea and the shipped behavior.
6. Do not edit code. If fixes are needed, summarize them as user-facing issues suitable for planning or triage.

Output:
- User journeys checked
- What works
- Gaps or risks
- Recommended next tasks, if any
