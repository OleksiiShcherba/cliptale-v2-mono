# CONTEXT — scene-generation-reference-gate

> Feature-local glossary. Roles and terms here are canonical for this feature's spec and downstream stages.
> Inherits the canonical terms of [storyboard-reference-flows/CONTEXT.md](../storyboard-reference-flows/CONTEXT.md)
> (Creator, Draft, Reference block, Reference flow, Character, Environment, Cast extraction, Cast confirmation,
> Starred result, Primary starred result, Scene link, Reference boundary, Scene generation master, No-flow state) —
> they keep their meaning here unchanged **except** where redefined below (the gate).

## Glossary

- **Creator** — the signed-in user who owns the draft and starts scene generation (same single acting role as in the inherited features).
- **Scene block** — a block on the Video Road Map (Step 2) representing one scene of the storyboard; its preview is produced by **scene generation**. NOT a reference block.
- **Scene generation** — the illustration/video generation that produces a scene block's preview (Step 3), started full-draft by the Creator from the canvas toolbar or per-scene by regenerating one scene block. Consumes **selected reference outputs** of the scene's linked reference blocks.
- **Ready reference block** — a reference block that has **≥1 completed result available to feed scene generation**. For a rolling-window-generated block this means its first generation finished (`window_status = done`) and produced an output; for a **manually-added block** (no rolling-window state) it means its linked reference flow produced ≥1 result. A block that is still pending/running its first generation, that failed with no usable result, or that has produced nothing is NOT ready. Replaces the old "block has ≥1 starred result" readiness notion.
- **Reference-done gate** (replaces the inherited **Star gate**) — the readiness rule for starting scene generation: the **full-draft** start may begin only when EVERY character AND environment reference block of the draft is a **ready reference block** AND — in a draft that has ≥1 reference block — every scene has ≥1 reference block linked to it; until then the action is rejected and the blocking blocks (and any reference-less scenes) are named with a recovery hint (retry or remove the block via the existing reference-flow controls, or link a reference to the scene). Regenerating an **individual scene** requires readiness only of the reference blocks **linked to that scene** (scene-linked scope). A draft with **zero** reference blocks passes the gate (scenes generate per the no-linked-blocks rule); the "every scene needs a link" rule applies only once a draft holds at least one reference block. Generation-complete — NOT starring — is the mandatory condition. The gate is enforced **server-side** at the start operation (its rejection is authoritative); any pre-emptive disabling of the start control in the UI is an inherited convenience, not re-specified here.
- **Selected reference output** — the **single** image of a ready reference block that feeds scene generation for a scene linked to that block: the block's **primary starred result** if one is set *and that result is a completed usable output*, otherwise the **deterministic default** = the **latest completed output** of that block. Starring therefore SELECTS which output is used; it no longer gates whether generation may start, and it no longer passes *multiple* candidates — the inherited multi-candidate "reference candidates" model (where the scene-generation master chose among several starred images per block) is **retired**: exactly one output per linked block reaches a scene. Guarantees a ready, linked block is never reference-less.
- **Reference-generation context** — the execution context that *produces* reference-block outputs: the media-worker rolling-window worker (and, for manually-added blocks, the reference-flow that backs them). The Reference-done gate reads readiness from the **persisted block state/output** this context writes (output-existence) — it does **not** subscribe to or await a live event/"completion signal"; "signalled completion" in the spec means "the persisted output exists at gate-evaluation time." A block whose first generation is still in progress simply has no persisted output yet and reads as not-ready.
- **Principal image** (a.k.a. general reference; *deprecated by this feature*) — the legacy single up-front reference image that scene generation used to require (stored in `storyboard_illustration_references`, approved via the principal-image modal). This feature **retires** it: it is no longer generated, approved, or read into scene generation; scenes consume per-cast **selected reference outputs** instead.
- **Blocking block** — a non-ready character/environment reference block named in the gate's rejection so the Creator knows exactly which block to finish, retry, or remove.
