## Findings (all resolved: **Fix now**)

### Blockers

**GAP-1 — AC-02 happy-path chain severed: cast-extraction never enqueued by the pipeline.**
`apps/media-worker/src/jobs/storyboardPipelineHooks.ts:181-204` (`onSceneGenerationComplete`) sets `reference_data → running` + `active_run_phase='reference_data'`, but nothing enqueues a cast-extract job. The only `enqueueCastExtract` caller is the inherited `apps/api/src/services/storyboardReference.extraction.service.ts:93`, which the pipeline never invokes; `storyboardPlan.job.ts:286-293` only calls the status-flip hook. Result: `reference_data` sits `running` until the reaper fails it at 10 min; `onCastProposalReady` never fires → AC-02 modal unreachable, blocking AC-03/AC-04 reachability. Test `storyboardPipelineHooks.integration.test.ts:142-146` only asserts the status flip (tautology). **Fix:** enqueue cast-extraction from scene-completion (mirror the autostart feature) + add a test asserting the job is enqueued.

**HB-1 — AC-12 inverted: heartbeat never refreshed during an in-progress phase → reaper kills healthy long runs.**
`apps/api/src/repositories/storyboardPipeline.repository.ts:244` (`recordHeartbeat`) has zero production callers. `heartbeat_at` is stamped only at `claimRun` (`repository.ts:165`) and `casAdvance` transitions (`storyboardPipelineHooks.ts:144`). The `reference_image` rolling window (`ai-generate.referenceWindow.ts`) completes blocks one-at-a-time without transitioning or refreshing the heartbeat, so any `reference_image`/`scene_image` run exceeding the 10-min bound (`storyboardPipelineReaper.job.ts:30`) is marked `failed` mid-progress — contradicts ADR-0005 / the design comment at `repository.ts:240-243`. The reaper test (`storyboardPipelineReaper.integration.test.ts:149`) only proves a fresh heartbeat is spared. **Fix:** call `recordHeartbeat({draftId, phase})` from per-unit completion points (`ai-generate.referenceWindow.ts:158`, scene-image success/fail at `storyboardOpenAIImage.job.ts:~318`) + add a multi-block aging-heartbeat test.

### Major (contract / payload)

**GAP-2 — AC-04 scene-image offer carries no precomputed estimate.**
`storyboardPipelineHooks.ts:305-316` advances `scene_image → awaiting_review` without computing/persisting a scene-image `cost_estimate`; `computeSceneImageEstimate` (`cost.service.ts:105-112`) has zero production callers (dead). Spec AC-04 + openapi require the offer to show a precomputed estimate. **Fix:** compute + persist the scene-image estimate when transitioning to the offer.

**GAP-3 — AC-02/AC-03 cast-proposal modal payload (proposal + ref-image estimate) never persisted.**
`onCastProposalReady` (`storyboardPipelineHooks.ts:211-229`) sets `reference_data → awaiting_review` but writes neither `payload_json.cast_proposal` nor `cost_estimate`; openapi `awaiting_review_cast` example (`contracts/openapi.yaml:83-103`) shows both populated. The ref-image estimate is only computed at confirm-revalidation (`confirm.service.ts:195`), never before the modal. **Fix:** project the cast-extraction `proposal_json` into `payload_json.cast_proposal` and persist `computeReferenceImageEstimate` into `cost_estimate` in `onCastProposalReady`.

**GAP-4 — AC-07 skipPhase missing `awaiting_review` precondition; contract 422 not enforced.**
`apps/api/src/services/storyboardPipeline.lifecycle.service.ts:118-143` writes `skipped` unconditionally; openapi `skip` (`contracts/openapi.yaml:272-281`) mandates 422 `pipeline.not_awaiting_review` when not `awaiting_review`. A skip on `idle`/`running` silently succeeds → mis-marks a never-run phase as intentional decline, corrupting the AC-08 prerequisite distinction. **Fix:** gate on `phaseStatuses(row)[phase] === 'awaiting_review'`, else throw the gate error; add the non-awaiting test.

**GAP-5 — AC-03 no-body "confirm-as-shown" path always 422s.**
`contracts/openapi.yaml:142-151` declares the body optional ("omit it to confirm the proposal exactly as shown"); controller passes `cost_estimate ?? null` (`controller.ts:145`) and `revalidateEstimate` (`cost.service.ts:155-161`) throws on null → documented no-body confirm always fails. All confirm tests pass an explicit estimate, hiding the gap. **Fix:** treat a missing client estimate as confirm-as-shown (validate against persisted `cost_estimate`), add the no-body test.

### Minor

- **MIN-1** — `process.env` read outside `config.ts`: `storyboardPipeline.resume.service.ts:45` reads `APP_PIPELINE_STUCK_PHASE_BOUND_MINUTES` directly, violating the `config.ts:39` convention (worker side did it right). **Fix:** add the key to `apps/api/src/config.ts`.
- **MIN-2** — reaper `pool.execute` (prepared) with `INTERVAL ? MINUTE` (`storyboardPipelineReaper.job.ts:52`) vs api repo `pool.query` (`storyboardPipeline.repository.ts:267`); mysql2 prepared-INTERVAL fragility. **Fix:** align both on `pool.query`.
- **MIN-3** — reaper CAS no-clobber race only structurally asserted (`storyboardPipelineReaper.integration.test.ts:172-213`, self-admitted in comment). **Fix:** extract `releaseStuckPhase(pool, draftId, phase, staleVersion)` and assert `affectedRows === 0` on a stale version. (Mechanism IS genuinely proven in `storyboardPipelineHooks.integration.test.ts:231-255`.)
- **MIN-4** — concurrent double-confirm not exercised: `confirm.service.test.ts:285` calls `confirmCast` twice sequentially, not `Promise.all`. CAS is sound; test-depth gap. **Fix:** add the concurrent race assertion.
