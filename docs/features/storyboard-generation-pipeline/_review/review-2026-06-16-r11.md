## Problem solved

After every pipeline step (scene completion, cast confirm, skip, scene-image completion) the React
Flow canvas was NOT updating — it stayed at the initial START+END state even when scene blocks,
reference blocks, and illustrations were ready in the backend. Users saw an empty canvas after
completing or skipping any flow.

---

## Root causes found

| # | Root cause | Location |
|---|---|---|
| RC-1 | `confirmPipelineCast` callback had no `reloadStoryboard()` call | `StoryboardPage.tsx:561` |
| RC-2 | `skipPhase reference_data` callback had no reload | `StoryboardPage.tsx:568` |
| RC-3 | `skipPhase scene_image` callback had no reload | `StoryboardPage.tsx:582` |

| RC-6 | No observer for `scene_image=completed` (scene illustrations) | `StoryboardPage.tsx` (missing) |
| RC-7 | `fitView` only fired on initial React Flow mount; new nodes after reload didn't refit viewport | `StoryboardCanvas.tsx` |

## Fixes applied

### `StoryboardCanvas.tsx`

- Added `fitViewTrigger?: number` prop to `StoryboardCanvasProps`
- `InnerCanvas` imports `useEffect` and calls `fitView({ duration: 300, padding: 0.3 })` whenever
  `fitViewTrigger` increments (guarded by `if (!fitViewTrigger) return;` to skip initial render)

### `StoryboardPageWorkspace.tsx`

- Added `fitViewTrigger?: number` to props interface and forwarded to `StoryboardCanvas`

### `StoryboardPage.tsx`

- Added `const [fitViewTrigger, setFitViewTrigger] = useState(0);`
- `reloadStoryboard` now calls `setFitViewTrigger(t => t + 1)` after `await reload?.()` resolves,
  causing `InnerCanvas` to run `fitView()` after the render with new nodes
- Added three phase-completion observers (useRef + useEffect pattern):
  - `scene: running → completed` → `void reloadStoryboard()`
  - `reference_image: * → completed` → `void reloadStoryboard()`
  - `scene_image: * → completed` → `void reloadStoryboard()`
  - All observers skip the initial mount value (guard: `if (prev === undefined) { prev = curr; return; }`)
    so page-open with an already-completed phase does NOT trigger a spurious reload
- `ReviewCastProposalModal.onConfirm`: chained `.then(() => reloadStoryboard())` after
  `confirmPipelineCast` resolves
- `ReviewCastProposalModal.onSkip`: chained reload after `skipPhase reference_data`
- `SceneImageOfferModal.onSkip`: chained reload after `skipPhase scene_image`
- `SceneImageOfferModal.onAccept`: no immediate reload — phase observer fires when
  `scene_image=completed`

### `e2e/storyboard-generation-pipeline.spec.ts`

- `stubPipeline` now also stubs `GET /storyboards/:draftId` and
  `GET /storyboards/:draftId/reference-blocks` (stubbed to minimal start+end / empty) so all
  pipeline tests are fully self-contained and independent of real API latency
  (pre-existing flakiness: after 15+ sequential tests, the 16th real storyboard GET timed out)
- **Flow 8** (2 new tests):
  - "Scene blocks appear on canvas when scene phase transitions to completed" — stubs storyboard
    GET to toggle between start+end and start+2scenes+end; emits `scene=completed` realtime;
    verifies `[data-testid=scene-block-node]` count goes 0 → 2
  - "Cast confirm triggers canvas reload — storyboard GET fired after onConfirm" — verifies that
    clicking confirm in `ReviewCastProposalModal` fires a subsequent `GET /storyboards/:draftId`
    via `page.waitForRequest`
