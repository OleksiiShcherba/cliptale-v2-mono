## Findings (new this round)

### Minors

**r4-F1 — AC-12: scene_image whole-phase failure has no co-located Retry.**
`apps/web-editor/src/features/storyboard/components/StoryboardPlanControls.tsx:186-230` (`StoryboardIllustrationControls`) renders the failure as `role="alert"` but its only menu (`StoryboardStatusMenu`) is gated on `status==='completed'` (`:220`) — the `failed` branch has no Retry, unlike `StoryboardPlanControls` which has `showRetry = status==='failed'` (`:74,101`). `PipelineFailureBanner` deliberately excludes scene_image. The whole-phase scene_image retry exists only via the `StepCorners` "Scene Images" button (enabled when `active_run_phase==null`), while the inline copy says "Retry failed scenes from their block" (`:157`, per-scene guidance). AC-12 ("tell what failed **with the option to retry** — for every phase") is **literally met** (retry reachable via StepCorners) but the affordance is not co-located and is untested for this path.
**Fix:** add a Retry button to the `failed` branch of `StoryboardIllustrationControls`, wired to `onRegenerate` (already → `triggerPhase('scene_image')`), mirroring the scene-phase control; add a component test.

**r4-F2 — AC-03 / contract: `CastConfirmation.references` is a dead no-op.**
`contracts/openapi.yaml:509-519` + `confirmCastBodySchema` (`storyboardPipeline.controller.ts:52-57`) accept a `references` array, but the controller forwards only `clientEstimate` to the service (`:142-146`) and `confirm.service.ts` derives the cast solely from the stored proposal — `references` is ignored. `ReviewCastProposalModal` is read-only. No AC requires "adjust" (AC-03 is confirm-as-shown), so this is **not an AC gap**, but the contract+schema overstate capability.
**Fix:** remove `references` from openapi `CastConfirmation` and `confirmCastBodySchema` so the contract matches shipped reality (confirm-as-shown); keep openapi + code in the same commit.
