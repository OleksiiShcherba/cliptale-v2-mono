### Stage-2 (test quality)

**F2-mock — mocked F2 test 1 emits an unrealistic post-skip state**
`e2e/storyboard-generation-pipeline.spec.ts:385` — the test "Skipping cast dismisses modal
and transitions to scene_image offer directly" manually emits `scene_image=awaiting_review`
after the skip API call. Live behavior: `skipPhase(reference_data)` only cascades
`reference_image → skipped`; `scene_image` stays `idle` until the Creator manually triggers
it via the corner control (`sb-pipeline-edge.cjs` confirmed this). The mock is testing the
**offer-modal UI** in isolation (valid) but via an unreachable state path.

**Resolution: Not an issue** — the mock correctly exercises the offer modal component (which
IS reachable when reference_image completes naturally) and the real skip→text-only flow is
separately verified live (edge script). No AC gap. Record why: the F2 mock was intentionally
written as a UI component test, not a full-system integration test; the live edge test covers
the actual flow.
