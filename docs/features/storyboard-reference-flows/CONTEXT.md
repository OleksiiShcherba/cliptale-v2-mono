# CONTEXT — storyboard-reference-flows

> Feature-local glossary. Roles and terms here are canonical for this feature's spec and downstream stages.
> Inherits the canonical terms of [generate-ai-flow/CONTEXT.md](../generate-ai-flow/CONTEXT.md) (Creator, Generation flow, Flow canvas, Result block, Generate, Cost confirmation, General library) — they keep their meaning here unchanged.

## Glossary

- **Creator** — the signed-in user who owns the draft and its reference flows (same role as in generate-ai-flow; the only acting role for this feature).
- **Draft** — a storyboard document a Creator works on; its Step 2 is the **Video Road Map** canvas (scene blocks + music blocks + reference blocks).
- **Reference block** (storyboard block type `AI Generation Flow`) — a new block on the Video Road Map canvas representing one character or one environment; lives off-chain like a music block (no edge connections), linked 1:1 to its **reference flow**. Clicking it opens the linked flow; it shows the **primary starred result** as its preview. NOT a scene block and NOT a content/generation block inside a flow.
- **Reference flow** — a Generation flow auto-created (or created on manual block addition) for exactly one character or environment of one draft. A full, ordinary Generation flow (editable, listed on the Generate AI page with a draft badge). NOT the old single "principal image".
- **Character** — a recurring person/figure of the script that the cast extraction proposes; gets one reference block + one reference flow. NOT a one-off extra mentioned in a single scene.
- **Environment** — a recurring location/setting of the script, same mechanics as a character. NOT a per-scene background description.
- **Cast extraction** — the AI step that reads the draft's script/scene plan and proposes the list of characters and environments, the assignment of the Creator's uploaded reference images to them, and the initial block↔scene links. NOT a paid image generation.
- **Cast confirmation** — the Creator's review step over the cast extraction output (add/remove/rename entries, reassign reference images, correct the proposed scene links in place) that shows ONE aggregate cost estimate; confirming it is the single collective cost confirmation that auto-starts the first generation in every reference flow. Replaces the old principal-image approval modal.
- **Starred result** (star) — a result in a reference flow the Creator marked with the star checkbox on its result block in the flow canvas. Multiple results may be starred; all starred images of a block are its **reference candidates**. NOT a library favorite — the star is scoped to the flow/block.
- **Primary starred result** — the one starred result designated as the reference block's preview image on the storyboard canvas (defaults to the earliest-starred result by the Creator's star action among the currently starred ones; reassignable). Un-starring or deleting it returns the block to the "no preview" placeholder.
- **Scene link** — an individually selected association between one reference block and one scene block (multi-select of specific scenes — NOT a start/end range like music blocks). AI-proposed at cast extraction, manually correctable via the block's scene selector; the linked scenes are listed on/under the block.
- **Reference boundary** — the domain invariant: when generating the preview of scene X, only the starred images of reference blocks linked to X are candidate references; images of unlinked blocks are never used for X. Within that boundary, the **scene generation master** decides which candidates to actually pass to the model.
- **Scene generation master** — the existing automated scene-preview planner, extended: per scene it selects which reference candidates (within the reference boundary) feed the generation; for scenes with no linked blocks it generates from the scene prompt plus a derived textual style description.
- **Star gate** — the readiness rule replacing the old principal-image approval: generation of the draft's FULL scene-preview set may start only when EVERY reference block of the draft has ≥1 starred result; until then the action is disabled with an explanation of which blocks are missing stars. Regenerating an individual scene requires stars only from the blocks linked to that scene; a draft with zero reference blocks passes the gate (scenes generate per the no-linked-blocks rule).
- **No-flow state** — the state of a reference block whose linked flow was deleted (after the warning): the block stays on the canvas with no preview and no reference candidates until re-linked/recreated.
