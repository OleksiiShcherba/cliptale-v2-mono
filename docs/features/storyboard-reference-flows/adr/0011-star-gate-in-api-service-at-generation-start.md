# 0011 — Enforce the star gate in the API service layer at generation start

- **Status:** Accepted
- **Date:** 2026-06-06
- **Deciders:** Tech Lead (Oleksii) + design Socratic walk

## Context

Star gate (AC-08/AC-08b): повний набір сцен стартує лише коли КОЖЕН reference-блок драфта має ≥1 зірку; регенерація однієї сцени X вимагає зірок лише від блоків, лінкованих до X; драфт без блоків проходить гейт. Відмова мусить називати конкретні блоки без зірок. SPA показує стан, api вирішує, worker виконує — де авторитетна перевірка?

## Decision drivers

- Quality goal 1: гейт стоїть ДО будь-якого списання грошей чи enqueue.
- AC-04: gate-повідомлення називає блоки і дії-виходи (retry / delete).
- Безпека: фронтенд-перевірка обходиться прямим HTTP-запитом — не може бути єдиною.
- ADR-0004: уникаємо ситуацій «гроші списані, виконання відмовлено» (жодних рефандів).

## Considered options

1. **Єдина авторитетна перевірка в `storyboardIllustration.service` у момент запиту на старт генерації** — до списання/enqueue; SPA читає той самий стан для disabled-кнопки (лише UX); worker довіряє вже провалідованим джобам.
2. **Дубль-перевірка в api + worker перед кожною сценою** — закриває TOCTOU-вікно, але джоба, що вже списала гроші, може впасти на гейті всередині виконання — рефанд-питання, яких ADR-0004 уникає; подвійна логіка = подвійний дрифт.

## Decision outcome

**Chosen:** Option 1. Гейт — бізнес-правило старту, не виконання; перевірка до грошей зберігає інваріант «списали → виконуємо». TOCTOU-вікно (зірку зняли між перевіркою і стартом) прийнятне: генерація йде зі знімком зірок на момент старту.

## Consequences

**Positive**
- Один авторитетний шматок логіки, повторно використаний обома режимами (повний набір / сцена X) і SPA-станом.
- Жодних мід-флайт відмов після списання.

**Negative**
- TOCTOU-вікно: знімок зірок на момент старту може відстати від останньої дії Creator-а на секунди — задокументовано як прийняте.

**Neutral**
- Повідомлення гейта (список блоків + дії-виходи) — частина API-контракту, фіксується стадією api.
