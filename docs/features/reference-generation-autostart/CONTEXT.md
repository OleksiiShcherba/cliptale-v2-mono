# CONTEXT — reference-generation-autostart

> Feature-local glossary. Roles and terms here are canonical for this feature's spec and downstream stages.
> Inherits the canonical terms of [storyboard-reference-flows/CONTEXT.md](../storyboard-reference-flows/CONTEXT.md)
> (Creator, Draft, Reference block, Reference flow, Character, Environment, Cast extraction, Cast confirmation,
> Cost confirmation, Video Road Map) and [scene-generation-reference-gate/CONTEXT.md](../scene-generation-reference-gate/CONTEXT.md) —
> they keep their meaning here unchanged **except** where redefined below.

## Glossary

- **Creator** — the signed-in user who owns the draft and starts reference generation (same single acting role as in the inherited features).
- **Step 1** — the earlier draft-editing step the Creator works through before the Video Road Map (script / scene-plan stage). NOT the Video Road Map. The precise content of Step 1 is out of this feature's scope; it matters here only as the step the Creator leaves when entering Step 2.
- **Step 2 (Video Road Map)** — the storyboard canvas step (scene blocks + music blocks + reference blocks), inherited unchanged. The step the Creator **enters** to trigger reference-generation auto-start.
- **Cast extraction** — the **free** AI step that reads the draft's script/scene plan and proposes the list of characters and environments, the assignment of uploaded reference images, and the initial block↔scene links (inherited; NOT a paid image generation). This is what **auto-starts** on Step-2 entry.
- **Cast confirmation** — the Creator's review step over the cast-extraction proposal, shown in the **Cast confirmation modal**, which carries ONE aggregate **Cost confirmation**; confirming it is the single collective consent that starts the **paid** first generation in every reference flow (inherited unchanged). This feature does NOT auto-confirm it — paid generation still requires explicit Creator consent.
- **Credits** — the account unit of charge spent on paid generation, surfaced as the aggregate cost estimate in the Cost confirmation. Free cast extraction spends zero credits; credits are debited only after the Creator confirms the cost. (The canonical unit referenced by the inherited Cost confirmation; named here so the spec's "no credits charged" phrasing is glossary-grounded.)
- **Reference-generation auto-start** — the new behavior introduced by this feature: when a Creator enters Step 2 for a draft that has no cast extraction yet, the system starts the free cast extraction **automatically and silently** (without forcing any modal open), so the cast proposal is already in progress or ready by the time the Creator opens the Cast confirmation modal. It fires **once per entry** and is a **no-op** when a cast extraction already exists for the draft (it never re-runs or duplicates). **"Entry" is each Step-2 open/re-mount/re-focus**, guarded on the draft's **persisted** extraction state: every open re-checks existence and auto-starts only when none exists (dedup keys on persisted state, not a session flag). A **failed auto-start** is one where the extraction request was **never accepted** (no extraction was created); a created-but-errored extraction counts as *existing* (no-op), not failed. It never charges credits and never bypasses the Cost confirmation.
- **Cast confirmation modal** — the modal surface that shows the cast-extraction proposal and the Cost confirmation, and from which the Creator confirms (paid) or cancels. It is a **proper dialog** (backdrop overlay, centered container, `dialog` semantics), consistent with the other storyboard modals. Its **pre-proposal state** (extraction still running, or not yet started) shows in-modal progress, NOT bare unstyled buttons rendered into the page body. Its **completed-empty state** (extraction finished proposing no characters/environments) shows a distinct "nothing to generate references for" surface with a close action and no confirm — empty-complete is *ready*, not still-in-progress.
- **Start reference generation control** — the manual affordance in the canvas toolbar that opens the Cast confirmation modal / triggers extraction. Retained after this feature as a **manual re-trigger / recovery** path (e.g. when auto-start failed or the Creator dismissed the modal); it is no longer the only way the extraction begins. It **always opens the Cast confirmation modal** in every entry case (not only after a failure), and **never starts a second extraction** when one already exists — it surfaces the existing one.
- **Stray-buttons defect** — the defect this feature removes: in the no-proposal state the Cast confirmation modal rendered two unstyled buttons ("Start reference generation" + "Cancel") inline at the bottom of the page, visually indistinguishable from the Step-2 footer, instead of a real modal.
