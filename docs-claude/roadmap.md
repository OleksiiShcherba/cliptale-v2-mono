## Product
**ClipTale** — AI-assisted web video editor. Users upload media, arrange clips on a timeline, add captions / AI-generated images, audio, and text overlays, preview live via Remotion, and export rendered MP4 videos. Background workers handle ingest (probe/thumbnail), transcription, AI asset generation (fal.ai + ElevenLabs), and final render (Remotion + Chromium).

## Small Domains (inline)

### Render Worker — `apps/render-worker/`
- **Purpose:** Consume BullMQ `render` jobs, run `@remotion/renderer` against `remotion-comps/VideoComposition`, upload MP4 to S3, update `render_jobs` row.
- **Key files:**
  - `src/index.ts` — BullMQ Worker bootstrap, subscribes to `render` queue
  - `src/config.ts` — env: `APP_REDIS_URL`, `APP_S3_*`, `APP_CHROMIUM_EXECUTABLE_PATH`
  - `src/jobs/render.job.ts` — job handler: fetch project → resolve clips → `renderMedia()` → upload
  - `src/lib/remotion-renderer.ts` — Remotion bundle + `renderMedia()` wrapper
  - `src/lib/db.ts` — MySQL pool, updates `render_jobs.status/result_url`
  - `src/lib/s3.ts` — upload with AWS SDK v3
- **Runs in:** Docker container with Chromium at `/usr/bin/chromium` (Puppeteer sandbox off — in-container only).
- **Input payload:** `RenderVideoJobPayload` from `packages/project-schema`.

### packages/project-schema
- **Purpose:** Canonical ProjectDoc shape — source of truth for timeline state across api, web-editor, workers.
- **Exports:** `projectDocSchema`, `trackSchema`, `clipSchema` (+ `videoClipSchema`, `audioClipSchema`, `textOverlayClipSchema`, `imageClipSchema`, `captionClipSchema`), type `ProjectDoc`, `Track`, `Clip`, plus job payload types (`MediaIngestJobPayload`, `TranscriptionJobPayload`, `RenderVideoJobPayload`, `CaptionWord`, `CaptionSegment`, `RenderPresetKey`, `RenderPreset`).
- **Structure:** `src/schemas/` (Zod), `src/types/` (derived TS types + job payloads).
- **Convention:** When changing any clip/track/project shape, update Zod schema first — all other packages consume via `@ai-video-editor/project-schema`.

### packages/api-contracts
- **Purpose:** OpenAPI spec + AI model catalogs shared between web-editor and api.
- **Key files:** `openapi.ts`, `fal-models.ts` (FAL_MODELS + capability groups), `elevenlabs-models.ts` (ELEVENLABS_MODELS + AUDIO_CAPABILITY_TO_GROUP).
- **Export:** `AI_MODELS` (union catalog), type `AiModel = FalModel | ElevenLabsModel` (discriminated by `provider`).

### packages/editor-core
- **Purpose:** Pure timeline helpers — **no React, no I/O**. One file: `src/index.ts`.
- **Current API:** `computeProjectDuration(clips, fps, minSeconds)` — guarantees non-zero duration for Remotion `<Player>`.

### packages/remotion-comps
- **Purpose:** All Remotion React compositions. Consumed by web-editor `<Player>` and by render-worker bundler.
- **Structure:**
  - `compositions/VideoComposition.tsx` + `.utils.ts` — top-level Composition rendering all tracks/clips
  - `layers/` — `VideoLayer`, `AudioLayer`, `ImageLayer`, `TextOverlayLayer`, `CaptionLayer`
  - `hooks/useRemotionEnvironment.ts` — distinguishes in-browser vs render
  - `remotion-entry.tsx` — registerRoot() entry for server render
  - `stories/` — Storybook stories (local-only)

### packages/ui
- **Purpose:** Reserved for shared UI primitives across apps. Currently thin (`src/index.ts` only) — most UI lives in `apps/web-editor/src/shared/components/`.

## AI Generation Flow (end-to-end)

```
web-editor ai-generation feature
  → POST /ai/generate   (apps/api/src/controllers/aiGeneration.controller.ts)
  → aiGeneration.service.ts  — validates via falOptions.validator / voiceCatalog
  → aiGenerationJob.repository  — INSERT ai_generation_jobs (status=queued)
  → enqueue-ai-generate.ts  — BullMQ push to `ai-generate` queue
  ───────────────────────────────────────────────────────────
  → media-worker/jobs/ai-generate.job.ts   (fal.ai image/video/text)
     OR
  → media-worker/jobs/ai-generate-audio.handler.ts  (ElevenLabs audio/voice)
  → fal-client.ts / elevenlabs-client.ts  — provider SDK calls
  → upload result to S3 → UPDATE ai_generation_jobs (status=done, result_url)
  ───────────────────────────────────────────────────────────
  → web-editor polls GET /ai/generate/:id → status
  → on done: asset is available, added to project (asset-manager feature)
```

**Audio is a separate ElevenLabs path** — NOT part of the fal.ai catalog. Unified `/ai/generate` endpoint, shared `ai-generate` queue. Speech-to-Speech accepts an `audio_upload` field.

## Render Flow

```
web-editor export feature
  → POST /renders  (renders.controller)
  → render.service (preset resolve, fixtures, list/create)
  → render.repository  — INSERT render_jobs (status=queued)
  → enqueue-render → BullMQ `render` queue
  → render-worker/jobs/render.job.ts
     → remotion-renderer.ts → @remotion/renderer renderMedia()
     → upload MP4 to S3 → UPDATE render_jobs.result_url
  → web-editor polls GET /renders/:id
```

## Ingest + Transcribe Flow

```
web-editor uploads asset  → presigned S3 PUT
  → POST /assets/finalize  (assets.controller → asset.finalize.service)
  → enqueue-ingest → BullMQ `ingest` queue
  → media-worker/jobs/ingest.job.ts  (ffprobe + thumbnails)
  → if video/audio: enqueue-transcription → `transcription` queue
  → media-worker/jobs/transcribe.job.ts  (OpenAI Whisper)
  → caption_tracks rows written
```

## Agent Navigation Guide

**Development workflow:** All testing goes through Docker Compose — not bare localhost. See `project_dev_workflow` memory.

**To add a new REST endpoint:**
1. Contract in `packages/api-contracts/src/openapi.ts` if public
2. `apps/api/src/routes/<domain>.routes.ts` — register route
3. `controllers/<domain>.controller.ts` — parse/validate with Zod middleware
4. `services/<domain>.service.ts` — business logic, never touches `req/res`
5. `repositories/<domain>.repository.ts` — raw SQL via `mysql2` pool from `db/connection.ts`
6. Wire router in `apps/api/src/index.ts`
7. Write co-located `*.test.ts` (Vitest)

**To change the timeline/clip shape:** Update `packages/project-schema/src/schemas/` first — web-editor, api, and both workers all consume this.

**To add an AI model/capability:** Extend `packages/api-contracts/src/fal-models.ts` or `elevenlabs-models.ts`; update `falOptions.validator` or `voiceCatalog.service` in api; add handler branch in `media-worker/jobs/ai-generate*.ts`.

**To add a new background job type:**
1. Define payload type in `packages/project-schema/src/types/job-payloads.ts`
2. Producer: `apps/api/src/queues/jobs/enqueue-<name>.ts` + register queue in `queues/bullmq.ts`
3. Consumer: `apps/media-worker/src/jobs/<name>.job.ts` (or render-worker)
4. Write `<name>.job.test.ts`

**To change DB schema:** Add next-numbered SQL file to `apps/api/src/db/migrations/`. Runs on container init — recreate the `db` volume locally (`docker compose down -v`) to re-seed.

**To add a web-editor feature:** Create `apps/web-editor/src/features/<name>/` with `components/`, `hooks/`, and feature-local state hooked into `store/project-store.ts` or `store/ephemeral-store.ts`. Use `lib/api-client.ts` for backend calls.

**To modify Remotion playback:** `packages/remotion-comps/src/compositions/VideoComposition.tsx` — same bundle used by browser `<Player>` AND server `renderMedia()`. Don't use DOM-only APIs inside layers.

**Critical invariants:**
- `apps/*/src/config.ts` is the ONLY place that reads `process.env`
- All errors from services MUST be typed (`lib/errors.ts`) — central handler maps them
- Integration tests hit a real MySQL — never mock the database (see `feedback_integration_tests` rule)
- Architecture/product decisions → stop and ask user (see `feedback_escalate_architecture` memory)
